
# 开发规范指南
为保证代码质量、可维护性、安全性与可扩展性，请在开发过程中严格遵循以下规范。

## 一、技术栈要求

- **主框架**：JDK 11
- **语言版本**：Java 11
- **核心依赖**：
  - `fastjson2`
  - `lombok`

## 二、目录结构

```
uav-common
└── src
    └── main
        └── java
            └── com
                └── ramble
                    └── uavcommon
                        └── demos
                            └── dto
```

## 三、SDK版本

- JDK 版本：11
- Maven 版本：使用项目中默认配置的 Maven 版本

## 四、构建工具

- 构造工具：Maven

## 五、依赖信息总结

- `fastjson2`：用于 JSON 序列化和反序列化，版本为 2.0.53。
- `lombok`：用于简化 Java 代码，减少样板代码，版本为 1.18.38。

## 六、代码作者

- 代码作者：cml

## 七、注释规范

- 所有类、方法、字段需添加注释，使用用户第一语言（中文）。

## 八、分层架构规范

| 层级        | 职责说明                         | 开发约束与注意事项                                               |
|-------------|----------------------------------|----------------------------------------------------------------|
| **Service**    | 实现业务逻辑、事务管理与数据校验   | 返回 DTO 而非 Entity（除非必要）                               |
| **DTO**      | 数据传输对象                       | 用于前后端数据传输，包名统一为 `demos.dto`                     |

### 接口与实现分离

- 所有接口实现类需放在接口所在包下的 `impl` 子包中。

## 九、安全与性能规范

### 输入校验

- 使用 `@Valid` 与 JSR-303 校验注解（如 `@NotBlank`, `@Size` 等）
  - 注意：JSR-303 校验注解位于 `javax.validation.constraints.*`

### 事务管理

- `@Transactional` 注解仅用于 **Service 层**方法。
- 避免在循环中频繁提交事务，影响性能。

## 十、代码风格规范

### 命名规范

| 类型       | 命名方式             | 示例                  |
|------------|----------------------|-----------------------|
| 类名       | UpperCamelCase       | `UserServiceImpl`     |
| 方法/变量  | lowerCamelCase       | `saveUser()`          |
| 常量       | UPPER_SNAKE_CASE     | `MAX_LOGIN_ATTEMPTS`  |

### 注释规范

- 所有类、方法、字段需添加注释，使用中文。

### 类型命名规范（阿里巴巴风格）

| 后缀 | 用途说明                     | 示例         |
|------|------------------------------|--------------|
| DTO  | 数据传输对象                 | `UserDTO`    |
| Util | 工具类                       | `StringUtils`|

### 实体类简化工具

- 使用 Lombok 注解替代手动编写 getter/setter/构造方法：
  - `@Data`
  - `@NoArgsConstructor`
  - `@AllArgsConstructor`

## 十一、扩展性与日志规范

### 接口优先原则

- 所有业务逻辑通过接口定义（如 `UserService`），具体实现放在 `impl` 包中（如 `UserServiceImpl`）。

### 日志记录

- 使用 `@Slf4j` 注解代替 `System.out.println`

## 十二、编码原则总结

| 原则       | 说明                                       |
|------------|--------------------------------------------|
| **SOLID**  | 高内聚、低耦合，增强可维护性与可扩展性     |
| **DRY**    | 避免重复代码，提高复用性                   |
| **KISS**   | 保持代码简洁易懂                           |
| **YAGNI**  | 不实现当前不需要的功能                     |
| **OWASP**  | 防范常见安全漏洞，如 SQL 注入、XSS 等      |
